# Changelog

All notable changes to judge-audit-mcp are documented here.

The format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and
this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.0] — 2026-07-15

Initial release.

### Added

**Core abstraction**
- `JudgeFingerprint` — model + prompt hash + rubric hash + scale + temperature +
  provider, with `comparable_to()` returning `identical | unknown | incomparable`.
  The third axis of the trilogy: `VersionRange.status_at()` (mobile-docs-mcp),
  `PolicyWindow.status_on()` (store-preflight-mcp), `JudgeFingerprint.comparable_to()`.
  Prompt and rubric are hashed, never stored; whitespace-normalised so a
  reformatted prompt isn't reported as a changed one.
- `temper_confidence()` — statistical findings downgrade themselves to `heuristic`
  below the power floor (30 paired obs, 50 for κ) and never gate a build. Direct
  analogue of `PolicyWindow.severity_on()`. Mechanical findings are exempt: a
  string comparison needs no sample size.

**Tools**
- `detect_judge_drift` — fingerprint diff + anchor-based decomposition of a score
  delta into judge shift and system shift. Anchors are items whose output is
  byte-identical across runs, auto-detected or declared. Without anchors, refuses
  to apportion rather than assuming the answer.
- `calibrate_judge` — Cohen's κ / weighted κ with bootstrap CI, reported next to
  the majority-class baseline and the inter-human ceiling.
- `bias_probe` — six probes: `position`, `verbosity`, `self_preference`,
  `length_confound`, `sycophancy`, `distribution`.
- `emit_probe_set` — generates the variant file a probe needs your judge to score.
- `audit_judge` — runs every check the supplied data supports; names what's missing
  for the rest.
- `explain_metric` — κ, anchors, fingerprints, effective levels, power, and why
  each is the right question.

**CLI**
- `judge-audit drift|calibrate|probe|emit` with `--fail-on` and text/JSON/SARIF
  output. SARIF renders as inline PR annotations.

**Statistics** (pure stdlib, no numpy/scipy)
- Cohen's κ, linear/quadratic weighted κ, chance and observed agreement, majority
  baseline, prevalence.
- Bootstrap percentile CIs over paired indices, Wilson score intervals, exact
  two-sided binomial and sign tests, paired Cohen's d.
- OLS via normal equations with bootstrap coefficient CIs; R².
- Entropy, effective levels (perplexity), ceiling/floor share, unused levels.
- No t-distribution or normal approximation anywhere: judge scores are bounded,
  discrete and lumpy, which is exactly where the textbook approximation
  embarrasses you.

**Optional live adapter** — `JUDGE_AUDIT_LIVE=1` plus an Anthropic or OpenAI key.
Off by default; the offline path is the product.

**Fixtures** — generated by `fixtures/generate.py` from stated mechanisms, seeded
and reproducible. Every scenario ships with a control built by the same generator
with the bias mechanism removed:
- `drift/` — the +6.0% headline that is 85% judge; plus a same-judge control, and
  a no-fingerprint variant.
- `calibration/` — κ=0.381 at 83% accuracy against a 78.5% baseline; plus a
  well-calibrated judge, and a noisy-humans variant where the judge is at the
  ceiling and the *task* is the problem.
- `self-preference/` — a claude judge favouring claude by +0.92 vs a human panel,
  where the claude outputs are also genuinely slightly better; plus an uninvolved
  gemini judge.
- `probes/` — position (26% contradictions, 81% first-slot skew), verbosity,
  sycophancy, collapsed/healthy/constant distributions, and the length-confound
  pair: two judges with near-identical naive correlations (r=0.82 vs r=0.78) and
  opposite controlled verdicts.
- `messy/` — nested JSONL, foreign CSV column names, wrapped JSON arrays, and a
  file with no id that must fail loudly.

**Tests** — `smoke_test.py`, 171 assertions (160 without scipy, which skips the
cross-checks rather than failing). Cross-checks every statistic against
scipy/scikit-learn where installed. Asserts both that probes fire on the
pathological fixture *and* that they stay quiet on the control.

### Notes
- One runtime dependency: `mcp`.
- `[verify]` extra (scipy/sklearn/numpy) is needed only to doubt this package,
  not to use it.
